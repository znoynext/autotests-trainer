# Autotests Trainer

Учебный стенд для практики автотестов на C#.

Внутри есть все, что нужно для обучения:

- UI в браузере;
- авторизация;
- API;
- SQLite-база;
- готовый проект для автотестов на xUnit;
- подготовка для UI-тестов на Playwright.

Демо-доступ:

```text
login: admin
password: admin
```

## Быстрый старт

Для первого запуска нужен только **.NET 8 SDK**.

Скачать .NET 8 SDK:

```text
https://dotnet.microsoft.com/download/dotnet/8.0
```

После установки открой новый терминал и проверь:

```powershell
dotnet --version
```

Если видишь версию `8.x.x`, все готово.

## Скачать проект

Самый простой способ без Git:

1. Открой репозиторий:

```text
https://github.com/znoynext/autotests-trainer
```

2. Нажми **Code**.
3. Нажми **Download ZIP**.
4. Распакуй архив.
5. Открой распакованную папку.

В папке проекта должны быть файлы:

```text

test.cmd
install-ui.cmd
AutotestsTrainer.sln
```

## Запустить сайт

Запусти файл:

```text
start.cmd
```

Можно двойным кликом. Можно из терминала:

```powershell
.\start.cmd
```

Скрипт сам восстановит зависимости и запустит сайт.

Открой в браузере:

```text
http://localhost:5171
```

Войди:

```text
admin / admin
```

Остановить сайт:

```text
Ctrl + C
```

## Запустить автотесты

Запусти файл:

```text
test.cmd
```

Или из терминала:

```powershell
.\test.cmd
```

Это запустит все тесты командой:


## Подготовить UI-тесты

Этот шаг нужен только когда начнешь писать UI-тесты через Playwright.

Запусти:

```text

```

Скрипт установит браузер Chromium для Playwright.

## Что есть в приложении

После запуска сайт доступен здесь:

```text
http://localhost:5171
```

Страницы:

- `/` - главная;
- `/account/login` - вход;
- `/dashboard` - закрытая панель после входа;
- `/workitems` - задачи;
- `/api-console` - ручная проверка API.

API:

```http
GET /api/workitems
POST /api/workitems
POST /api/workitems/{id}/complete
```

База данных создается автоматически:

```text
src/AutotestsTrainer.Web/App_Data/autotests.db
```

Отдельно SQLite устанавливать не нужно.

## Где писать свои тесты

Папка тестового проекта:

```text
tests/AutotestsTrainer.Tests
```

Папка для UI-тестов:

```text
tests/AutotestsTrainer.Tests/Ui
```

Проект уже подключен к xUnit. Playwright тоже подключен как NuGet-пакет, но браузер для него ставится через `install-ui.cmd`.

## Какие тесты здесь тренировать

### Первые smoke-тесты

- сайт открывается;
- страница логина открывается;
- API отвечает;
- тестовый проект запускается.

### Тесты авторизации

- вход с `admin / admin` успешен;
- неправильный пароль не пускает;
- закрытая страница недоступна без входа;
- после выхода закрытая страница снова недоступна.

### API-тесты

- получить список задач;
- создать задачу;
- проверить JSON-ответ;
- закрыть задачу;
- проверить ошибку для несуществующего `id`;
- проверить ошибку для неправильных данных.

### UI-тесты

- открыть сайт в браузере;
- перейти на логин;
- заполнить логин и пароль;
- войти;
- создать задачу через форму;
- проверить, что задача появилась в списке;
- выйти.

### Проверки базы данных

- после создания задачи запись появилась в SQLite;
- после закрытия задачи статус изменился;
- данные из API совпадают с данными в базе.

## Рекомендуемый порядок обучения

1. Запусти сайт через `start.cmd`.
2. Войди руками как `admin / admin`.
3. Создай задачу через UI.
4. Открой `/api-console`.
5. Запусти тесты через `test.cmd`.
6. Напиши простой smoke-тест.
7. Напиши тест на логин.
8. Напиши API-тест на создание задачи.
9. Запусти `install-ui.cmd`.
10. Напиши UI-тест на вход через браузер.

## Частые проблемы

### `dotnet` не найден

Установи .NET 8 SDK, закрой терминал и открой новый.

Проверка:

```powershell
dotnet --version
```

### Сайт не открывается

Проверь, что окно `start.cmd` открыто и в нем нет ошибки.

Если порт занят, останови старый запуск:

```text
Ctrl + C
```

Потом снова запусти `start.cmd`.

### UI-тесты не запускают браузер

Запусти:

```text
install-ui.cmd
```

### Нужно сбросить базу

Останови сайт и удали файлы из папки:

```text
src/AutotestsTrainer.Web/App_Data
```

При следующем запуске база создастся заново.

## Что не хранится в GitHub

Репозиторий публичный. Локальные и временные файлы не добавляются.

- `bin/`;
- `obj/`;
- локальная SQLite-база `*.db`;
- временные файлы SQLite;
- логи;
- пользовательские файлы IDE.
