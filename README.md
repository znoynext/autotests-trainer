# Autotests Trainer

Учебный pet-проект для практики автотестов на C#.

В проекте есть локальный стенд с UI, авторизацией, API и SQLite-базой. Его можно запускать на своем компьютере и тренироваться писать автотесты без внешних сервисов.

Демо-доступ:

```text
login: admin
password: admin
```

## Самый простой запуск

Минимально нужно установить только **.NET 8 SDK**.

Скачать:

```text
https://dotnet.microsoft.com/download/dotnet/8.0
```

После установки открой новый PowerShell или cmd и проверь:

```powershell
dotnet --version
```

Если команда показывает версию `8.x.x`, можно запускать проект.

## Как скачать проект без Git

1. Открой репозиторий:

```text
https://github.com/znoynext/autotests-trainer
```

2. Нажми зеленую кнопку **Code**.
3. Нажми **Download ZIP**.
4. Распакуй архив в удобную папку.
5. Открой распакованную папку проекта.

## Как запустить сайт

В папке проекта запусти файл:

```text
start.cmd
```

Можно двойным кликом, можно из терминала:

```powershell
.\start.cmd
```

После запуска открой в браузере:

```text
http://localhost:5171
```

Войди:

```text
admin / admin
```

Чтобы остановить сайт, нажми в окне запуска:

```text
Ctrl + C
```

## Как запустить тесты

В папке проекта запусти:

```text
test.cmd
```

Или из терминала:

```powershell
.\test.cmd
```

Это выполнит:

```powershell
dotnet test
```

## Как подготовить UI-тесты

Это нужно только когда начнешь писать UI-тесты на Playwright.

В папке проекта запусти:

```text
install-ui.cmd
```

Или из терминала:

```powershell
.\install-ui.cmd
```

Скрипт соберет проект и установит браузер Chromium для Playwright.

## Что есть в проекте

После запуска доступен сайт:

```text

```

Внутри есть:

## Какие тесты можно писать

### UI-тесты

Через Playwright на C#.

Идеи:

- открыть главную страницу;
- перейти на страницу входа;
- войти как `admin / admin`;
- проверить, что открылась панель;
- создать задачу через форму;
- проверить, что задача появилась в списке;
- выйти из системы.

### API-тесты

Через `HttpClient` или `WebApplicationFactory`.

Эндпоинты:

```http
GET /api/workitems
POST /api/workitems
POST /api/workitems/{id}/complete
```

Идеи:

- получить список задач;
- создать задачу;
- проверить JSON-ответ;
- закрыть задачу;
- проверить несуществующий `id`;
- проверить неправильные данные.

### Тесты авторизации

Идеи:

- правильный логин пускает в систему;
- неправильный пароль не пускает;
- закрытая страница недоступна без входа;
- после выхода закрытая страница снова недоступна.

### Проверки базы данных

SQLite-база создается здесь:

```text
src/AutotestsTrainer.Web/App_Data/autotests.db
```

Идеи:

- после создания задачи запись появилась в БД;
- после закрытия задачи статус изменился;
- данные из API совпадают с данными в БД.

### Smoke-тесты

Самые первые проверки:

- сайт открывается;
- страница логина открывается;
- API отвечает;
- тестовый проект запускается.

## Рекомендуемый порядок обучения

1. Запусти сайт через `start.cmd`.
2. Руками войди как `admin / admin`.
3. Создай задачу через UI.
4. Открой API-консоль и посмотри ответы API.
5. Запусти `test.cmd`.
6. Напиши первый smoke-тест.
7. Напиши тест на логин.
8. Напиши API-тест на создание задачи.
9. Запусти `install-ui.cmd`.
10. Напиши UI-тест на вход через браузер.

## Частые проблемы

### `dotnet` не найден

Установи .NET 8 SDK:

```text
https://dotnet.microsoft.com/download/dotnet/8.0
```

После установки закрой терминал и открой заново.

### Сайт не открывается

Проверь, что `start.cmd` все еще работает и в окне нет ошибки.

Если порт занят, останови предыдущий запуск через:

```text
Ctrl + C
```

### UI-тесты не запускают браузер

Запусти:

```text
install-ui.cmd
```

### Нужно сбросить базу

Останови сайт и удали файлы из папки:

```text
src/AutotestsTrainer.Web/App_Data
```

При следующем запуске база создастся заново.

## Что не хранится в GitHub

Репозиторий публичный, поэтому локальные и временные файлы не добавляются.

В `.gitignore` исключены:

- `bin/`;
- `obj/`;
- локальная SQLite-база `*.db`;
- временные файлы SQLite;
- локальные логи;
- пользовательские файлы IDE.

В GitHub хранится только код, тестовая инфраструктура и документация.
